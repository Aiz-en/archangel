## Project Overview

**Archangel** is an automated day trading bot built in Python. Market data comes from `yfinance` (Yahoo Finance, free) during the paper-trading phase; the Webull Open API is reserved for live order execution and may later replace yfinance for streaming if we pay for the subscription. The current phase is building a **paper trading engine** that consumes real market data but simulates order fills locally.

## Architecture (Planned)

- **Market data layer**: Free historical & intraday OHLCV via `yfinance`. Real-time/streaming via Webull MQTT is deferred — requires a paid OpenAPI Advanced Quotes subscription (see `docs/webull_api_reference.md`).
- **Strategy engine**: Pure functions that take market data in and output trade signals
- **Paper trading engine**: Simulates order fills with configurable slippage, tracks positions and P&L
- **Persistence**: SQLite for trade logs, position history, and performance metrics
- **Live trading layer** (future): Replaces paper engine with real Webull order execution

## Webull Open API

- Python SDK: `pip3 install --upgrade webull-openapi-python-sdk`
- Protocols: HTTP (REST), MQTT (streaming market data), gRPC (trade event subscriptions)
- Auth: App Key + App Secret, SDK handles request signing
- Rate limits: orders 600/60s, previews 150/10s, account 2/2s, instruments 10/30s
- Docs: https://developer.webull.com/apis/docs/
- SDK repo: https://github.com/webull-inc/openapi-python-sdk

## Development

### Python version
- **Use Python 3.13** — the Webull SDK supports `>=3.8,<3.14`
- System Python on this Mac is 3.14 (too new), so the venv is built explicitly from `/usr/local/bin/python3.13`

### Virtual environment
- Located at `.venv/` (gitignored)
- **Activate in every new terminal session** before running anything:
  ```bash
  cd ~/archangel_proj
  source .venv/bin/activate
  ```
- Prompt shows `(.venv)` when active. Without activation, `python` falls back to system 3.14 and the SDK won't be found.

### Reproducing the env from scratch
```bash
/usr/local/bin/python3.13 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### Project layout
- `code_base/` — Python source
  - `hello_webull.py` — Webull auth smoke test (calls `trade_client.account_v2.get_account_list()`)
  - `hello_quote.py` — market-data smoke test, fetches 1m AAPL bars via yfinance
  - `practice.py` — Python learning scratchpad (not part of the bot)
- `docs/` — strategy spec and Webull API reference
- `.env` — Webull App Key/Secret (gitignored, present locally)
- `conf/token.txt` — auto-generated by Webull SDK to cache auth token (gitignored)
- Run scripts with the venv active: `python code_base/hello_quote.py`

- `code_base/screener.py` — live multi-criteria screener that maintains an in-memory watchlist for the trading algorithm. Two-stage: (1) **coarse pass** reuses `scanner.WebullScanner.get_top_gainers()` then filters %change/price/volume from the feed; (2) **fine pass** enriches each survivor via yfinance `.info` (`floatShares`, `averageVolume`, `marketCap`), computes RVOL = today's volume / avg volume, and applies the float-≤-20M and RVOL-≥-5x filters. Float/avg-vol are cached per symbol per day so the loop stays fast. `ScreenCriteria` holds all thresholds (defaults = the strategy's selection rules); `Candidate` is the enriched watchlist type and stable swap boundary. `LiveScreener.run()` is a blocking console dashboard; `LiveScreener.start()` spawns a daemon thread and exposes `.watchlist` for the algorithm to read. RTH-gated (`is_market_open`, ET, ignores holidays). **CLI**: `python code_base/screener.py` runs the live dashboard; `--once` prints a single screen, `--ignore-hours` screens after close (for testing), `--min-change/--price-min/--price-max/--min-volume/--max-float/--min-rvol/--rank-type` tune the criteria, `--smoke` verifies the pipeline. No SQLite persistence by design. Smoke test passes against live data (strict + relaxed screens render, enrichment verified on AAPL). **vs. `scanner.py`**: scanner = raw Webull gainer feed; screener = applies the full strategy criteria on top.

## Webull paper-trading setup (one-time)
1. Open a fresh isolated Webull account (no funded live side).
2. In the Webull app, enable paper trading.
3. Copy `.env.example` to `.env` and fill in `WEBULL_PAPER_USERNAME` / `WEBULL_PAPER_PASSWORD`.
4. Run `python code_base/webull_paper.py` — it will prompt for an MFA code and cache the session.
5. Subsequent runs skip MFA for ~30 days.
